　　这两年 Docker 比较火，很多大厂都在使用。这里记录一下使用 Docker 安装 Mysql 并连接的过程。
<!--more-->


### 一、安装 Mysql

#### 1、查找镜像

　　在 Docker Hub 官网查找自己需要的版本 [https://hub.docker.com/_/mysql?tab=tags](https://hub.docker.com/_/mysql?tab=tags)

#### 2、下载镜像

　　我选择的是 5.7.29 版本
```shell 
docker pull mysql:5.7.29
```

#### 3、创建项目文件夹

　　创建一个文件夹用来存放 Mysql 的配置文件、日志以及数据等（也就是挂载目录，作用是将此目录中的文件或文件夹覆盖掉容器内部的文件或文件夹）
```shell
mkdir -p /mnt/docker1/mysql
cd /mnt/docker1/mysql
```


### 二、运行 Mysql

#### 1、使用镜像创建容器并运行

```shell 
docker run -p 3306:3306 -d --name docker-mysql -v $PWD/conf:/etc/mysql/conf.d -v $PWD/logs:/logs -v $PWD/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=123456 mysql
```

指令解释：
- `-p` 表示端口映射，`-p 宿主机 port:容器 port`
- `-d` 表示后台运行并返回容器 id，只有在父进程即 docker 进程退出的时候才会去把容器退出，除非你使用了`--rm`选项。如果你在运行容器时将`-d`和`--rm`两个选项一起使用，那么容器会在退出或者后台进程停止的的时候自动移除掉（只要一个情况便会自动移除镜像）
- `--name` 表示给容器指定的名称
- `-v $PWD/conf:/etc/mysql/conf.d` 表示将主机上当前目录下的 conf/my.cnf 文件挂载到容器的 /etc/mysql/my.cnf
- `-v $PWD/logs:/logs` 表示将主机上当前目录下的 logs 目录挂载到容器的 /logs
- `-v $PWD/data:/var/lib/mysql` 表示将主机上当前目录下的 data 目录挂载到容器的 /var/lib/mysql 
- `-e MYSQL_ROOT_PASSWORD=123456` 表示初始化 root 用户的密码

#### 2、进入容器

``` shell
docker exec -it docker-mysql /bin/bash
```

#### 3、进入 Mysql

``` shell
mysql -uroot -p
```
